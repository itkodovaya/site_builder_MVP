# Apps

Точки входа приложений (API gateway/BFF, админка, воркеры).

## Назначение

В этой директории размещаются приложения, которые являются точками входа в систему:

- **API Gateway / BFF** - единая точка входа для клиентов, маршрутизация запросов к сервисам
- **Admin Panel Backend** - API для административной панели
- **Workers** - фоновые воркеры для обработки задач

## Правила интеграции сервисов

1. **Apps не содержат бизнес-логику** - они только маршрутизируют запросы к сервисам
2. **Сервисы независимы** - каждый сервис в `services/` может работать автономно
3. **Общение через контракты** - используйте `packages/shared_contracts/` для межсервисного взаимодействия
4. **Версионирование API** - версионируйте API через префиксы (`/v1/`, `/v2/`)

## Структура приложения

```
apps/
└─ <app_name>/
   ├─ src/
   │  ├─ routes/          # Маршруты
   │  ├─ middleware/      # Middleware
   │  ├─ clients/         # Клиенты для сервисов
   │  └─ main.ts          # Bootstrap
   ├─ test/
   ├─ Dockerfile
   └─ README.md
```

## Добавление нового приложения

1. Создайте директорию `apps/<app_name>/`
2. Следуйте структуре выше
3. Добавьте `README.md` с описанием назначения
4. Настройте Dockerfile для контейнеризации
