# ADR-001: Структура монорепо

## Status

Accepted

## Context

Необходимо организовать backend-код в монорепозитории для упрощения разработки и поддержки нескольких сервисов.

## Decision

Использовать монорепозиторий со следующей структурой:

```
backend/
├─ apps/              # Точки входа
├─ services/          # Микросервисы
├─ packages/          # Общие модули
├─ infra/             # Инфраструктура
├─ tools/             # Скрипты разработки
└─ docs/              # Документация
```

## Consequences

### Положительные

- Единая точка входа для разработки
- Проще управлять зависимостями между сервисами
- Общие модули доступны всем сервисам
- Единые стандарты кодирования

### Отрицательные

- Больший размер репозитория
- Необходимость координации изменений в общих модулях
- Потенциальные проблемы с масштабированием команды
